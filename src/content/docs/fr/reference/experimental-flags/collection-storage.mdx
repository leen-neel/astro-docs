  **Type :** `"single-file" | "chunked"`<br />
  **Par défaut :** `"single-file"`<br />
  <Since v="7.1.0" />

Cette fonctionnalité expérimentale vous permet de modifier la manière dont les données des collections de contenu sont conservées.

Par défaut, Astro conserve ces données dans un seul fichier à l'emplacement `.astro/data-store.json`. Lorsque l'option est définie sur `"chunked"`, Astro répartit les données entre plusieurs fichiers dans le dossier `.astro/data-store/`.

```js title="astro.config.mjs" ins={5}
import { defineConfig } from "astro/config";

export default defineConfig({
  experimental: {
    collectionStorage: "chunked"
  }
});
```

En mode `chunked`, les fichiers sont répartis en plusieurs fragments. Astro crée un nouveau fragment lorsque le fragment actuel dépasse 10 Mo.

Cette option est recommandée lors du déploiement sur des plateformes imposant une limite de taille pour les ressources individuelles.
