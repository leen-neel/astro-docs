<p>

  **타입:** `"single-file" | "chunked"`<br />
  **기본값:** `"single-file"`<br />
  <Since v="7.1.0" />
</p>

이 실험적 기능을 사용하면 콘텐츠 컬렉션 데이터를 저장하는 방식을 변경할 수 있습니다.

기본적으로 Astro는 이 데이터를 `.astro/data-store.json`에 단일 파일로 저장합니다. `"chunked"`로 설정하면 Astro는 데이터를 `.astro/data-store/` 폴더 안의 여러 파일로 나누어 저장합니다.

```js title="astro.config.mjs" ins={5}
import { defineConfig } from "astro/config";

export default defineConfig({
  experimental: {
    collectionStorage: "chunked"
  }
});
```

청크 모드에서는 데이터가 여러 청크 파일로 분할됩니다. 현재 청크의 크기가 10MB를 초과하면 Astro는 새로운 청크를 생성합니다.

이 옵션은 개별 자산의 크기에 제한이 있는 플랫폼에 배포하는 경우 사용하는 것이 좋습니다.
